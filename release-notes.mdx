<Update label="November 17–23, 2025" tags={["Kubernetes clusters", "Applications"]}>
  - _Added:_ Managed Kubernetes now performs health checks on nodes, and automatically recovers nodes that are not ready or have issues with the boot disk or GPUs.
  - _Added:_ Four [NVIDIA NIM](https://www.nvidia.com/en-us/ai-data-science/products/nim-microservices/) microservices are now available as standalone applications. These microservices provide models from the NVIDIA BioNeMo Framework for computational drug discovery: [Boltz-2](https://build.nvidia.com/mit/boltz2/modelcard), [Evo 2](https://build.nvidia.com/arc/evo2-40b/modelcard), [GenMol](https://build.nvidia.com/nvidia/genmol-generate/modelcard) and [MolMIM](https://build.nvidia.com/nvidia/molmim-generate/modelcard). [Learn more](https://docs.nebius.com/applications/standalone/nvidia-nim).
  - _Added:_ The [NVIDIA Blueprint for generative virtual screening](https://github.com/NVIDIA-BioNeMo-blueprints/generative-virtual-screening) is now available as an application for Managed Kubernetes. The blueprint provides an AI-driven pipeline to design optimized small molecules efficiently. [Learn more](https://docs.nebius.com/applications/kubernetes/nvidia-blueprint-virtual-screening).
</Update>

<Update label="November 10–16, 2025" tags={["Virtual machines", "Billing"]}>
  - _Updated:_ You can now only delay a [maintenance event for Compute virtual machines](https://docs.nebius.com/compute/virtual-machines/maintenance) two times, up to seven days each time.
  - _Added:_ For residents of Texas and Washington, US, Nebius now [calculates taxes](https://docs.nebius.com/signup-billing/payments/taxes#usa) in accordance with United States tax law and includes them in invoices.
</Update>

<Update label="November 3–9, 2025" tags={["Virtual machines", "Kubernetes clusters"]}>
  - _Updated:_ The default quota on the number of NVIDIA H200 GPUs in the `us-central1` region per tenant is now 16, down from 32. The change does not affect existing tenants. See [all Compute quotas](https://docs.nebius.com/compute/resources/quotas-limits).
  - _Updated:_ When you [manually install the NVIDIA GPU Operator](https://docs.nebius.com/kubernetes/gpu/set-up#drivers-install) on your InfiniBand-interconnected nodes, GPUDirect RDMA is now enabled by default.
</Update>

<Update label="October 27–November 2, 2025" tags={["PostgreSQL clusters"]}>
  - _Added:_ Monitoring dashboards for Managed PostgreSQL clusters now include two new [connection pooler metrics](https://docs.nebius.com/postgresql/monitoring#pooler-metrics), **Pooler clients active** and **Pooler clients waiting**.
</Update>

<Update label="October 20–26, 2025" tags={["Kubernetes clusters"]}>
  - _Added:_ Managed Kubernetes now supports Kubernetes 1.32. This version is selected for new clusters and node groups by default. See the [overview of Kubernetes versions](https://docs.nebius.com/kubernetes/versions).
</Update>

<Update label="September 29–October 5, 2025" tags={["Virtual machines", "MLflow clusters"]}>
  - _Added:_ You can now create [preemptible virtual machines](https://docs.nebius.com/compute/virtual-machines/preemptible) with NVIDIA B200 GPUs in the `us-central1` region.


  - _Removed:_ New Managed MLflow clusters no longer support the `small` size. See the guide to [creating clusters](https://docs.nebius.com/mlflow/clusters/manage).
</Update>

<Update label="June 23–29, 2025" tags={["Virtual machines", "Disks and shared filesystems"]}>
  - _Updated:_ The size of a Compute virtual machine's boot disk is now limited to a maximum of 30 TiB. See the guide to [creating boot disks](https://docs.nebius.com/compute/storage/manage#boot).
</Update>

<Update label="February 10–16, 2025" tags={["PostgreSQL clusters"]}>
  - _Updated:_ The pgvector extension installed on Managed PostgreSQL clusters is upgraded from version 0.7.2 to 0.8.0. See [all available PostgreSQL extensions](https://docs.nebius.com/postgresql/databases/extensions#list).
</Update>
